TROUBLESHOOTING QUICK FIX GUIDE
================================================================================

PROBLEM: "Database locked" when running setup script
  FIX: Close all database connections and try again
  Commands:
    • Close backend if running
    • Close IDE database viewers
    • Delete .db-journal files if any
    • Run: python scripts/setup_clinician_assignment.py

PROBLEM: Test shows "Column missing"
  FIX: Migrations not applied yet
  Solution: Run setup script first
    python scripts/setup_clinician_assignment.py

PROBLEM: "Clinician still sees all accounts"
  FIX: Either (1) test data not set up, or (2) front-end cache issue
  Solution:
    1. Run setup and test scripts first
    2. Clear browser cache: localStorage.clear() in DevTools Console
    3. Refresh page: Ctrl+F5
    4. Re-login as clinician

PROBLEM: Mobile app shows "501 Not Implemented" for encryption
  FIX: Encryption key not in environment
  Solution: verify .env file has PHI_ENCRYPTION_KEY set
    PHI_ENCRYPTION_KEY=DV6bGaOXYP29VZmLHeyaitlwzZpcMJ2x1OjwoTE4tLA=

PROBLEM: Backend fails to start after migrations
  FIX: ML model load issue (unrelated to migrations)
  Solution: Check startup.log for details, restart backend

PROBLEM: Test 8 fails: "No assigned patients"
  FIX: Setup script didn't assign patients
  Solution: Run setup script again and check output for assignment verification
